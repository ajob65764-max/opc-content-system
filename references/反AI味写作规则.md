# 反 AI 味写作规则

这个文件记录“不要怎么写”。每次发现 Codex 出现你不喜欢的表达，都可以把它补充进来。

## 一、默认禁止的表达问题

### 1. 不要过度总结

避免每一小节都出现“总的来说”“综上所述”“由此可见”。内容已经说清楚时，不再重复总结一遍。

### 2. 不要空泛升华

避免没有事实支撑的句子，例如：

- “这不仅是一种方法，更是一种全新的生活方式。”
- “真正厉害的人，都懂得顺势而为。”
- “未来属于那些敢于改变的人。”

如果要写结论，必须回到具体问题、具体判断或具体动作。

### 3. 不要万能句式

减少以下机械结构：

- “不是……而是……”连续出现；
- “首先、其次、再次、最后”堆叠；
- “在这个快速发展的时代”；
- “随着科技的不断进步”；
- “赋能、闭环、抓手、底层逻辑”无必要堆砌。

这些词并非绝对不能使用，但必须真的有信息价值。

### 4. 不要夸大收益

不写“轻松月入过万”“普通人翻身机会”“零基础躺赚”“用了就能成功”等无法保证的结果。

### 5. 不要编造经历

用户没有提供的客户案例、收入数字、职业背景、使用感受和成功结果，不得虚构。缺少材料时先询问，或者明确写成假设示例。

### 6. 不要把简单问题包装复杂

能用3句话说清楚的内容，不扩成10个步骤。能直接给成品时，不只给方法论。

### 7. 不要强行营销

避免无关的私信引导、评论暗号、制造焦虑和虚假稀缺。需要转化时，行动引导必须与正文价值自然衔接。

## 二、默认写作要求

- 先说结论，再解释原因；
- 使用短句，每段只讲一个主要意思；
- 多写真实场景、具体动作和判断标准；
- 保留用户原有语气，不擅自拔高观点；
- 口播稿要像人说话，文章稿要像人写作；
- 不为了“专业”而堆砌行业术语；
- 不为了“完整”而添加用户不需要的内容。

## 三、出现以下表达时先检查

| 容易有AI味的表达 | 更自然的处理方式 |
| --- | --- |
| 值得注意的是 | 直接写需要注意的事情 |
| 不难发现 | 说明具体发现和依据 |
| 在某种程度上 | 明确适用条件 |
| 这无疑是 | 删除或给出证据 |
| 从本质上来说 | 直接写核心判断 |
| 让我们一起 | 改成明确动作 |
| 希望对你有所帮助 | 用具体下一步结束 |

## 五、收到修改意见后的处理

当我明确指出“这句话不要再写”“以后统一这样处理”时：

1. 先修改当前内容；
2. 判断这是不是长期规则；
3. 如果是长期规则，经过我确认后补充到本文件；
4. 不要把一次性的具体修改误写成永久禁令。
