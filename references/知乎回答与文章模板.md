# 知乎回答与文章模板

适用于经验型回答、方法论拆解、行业问题、职业判断、工具评测和观点论证。

## 一、开始前需要的信息

```text
原问题：
提问者真正想解决什么：
目标读者：
一句话答案：
主要理由：
个人经历或案例：
可引用的事实或资料：
需要说明的边界：
必须保留：
明确不要：
```

## 二、标准回答结构

### 1. 先正面回答

开头用1—3段给出明确结论。不要先写大段行业背景，也不要用“这个问题要分情况”回避回答。确实需要分情况时，先说明主要判断，再列条件。

### 2. 展开论证

推荐结构：

```text
明确结论
→ 为什么这样判断
→ 真实经历或案例
→ 具体方法
→ 反例或适用边界
→ 回到原问题
```

### 3. 区分信息类型

- 可以验证的信息：给出来源或说明依据；
- 个人经历：明确这是个人经验；
- 主观判断：说明适用条件，不包装成普遍事实。

### 4. 结尾

回到提问本身，给出读者下一步可以采取的动作。不写与问题无关的升华。

```text
目标读者：[填写]
一句话答案：[填写]
主要理由：[填写]
个人经历或案例：[填写]
事实资料：[填写]
适用边界：[填写]
必须保留：[填写]
明确不要：[填写]

请先读取 About_me 和本模板。输出：
1. 一段直接回答；
2. 完整论证；
3. 必要的小标题；
4. 事实、经验和判断的区分；
5. 简洁结尾；
6. 发布前检查结果。

不要堆砌正确但无用的观点，不编造数据和经历。完成后保存到 Outputs/知乎。
```

## 四、简短示例

问题：普通人有必要搭建自己的AI工作系统吗？

开头示例：

> 如果你只是偶尔让AI改一句话，没有必要专门搭系统。但如果你每周都在重复写内容、整理资料或推进副业项目，最小工作系统很有必要。它解决的不是“AI够不够聪明”，而是背景能不能复用、结果能不能保存、经验能不能留下。

## 五、发布前检查

- [ ] 开头已经正面回答问题；
- [ ] 论点有理由、案例或资料支撑；
- [ ] 事实、个人经验和主观判断已经区分；
- [ ] 没有为了显得专业而堆砌术语；
- [ ] 说明了适用边界；
- [ ] 结尾回到原问题；
- [ ] 已保存到 `Outputs/知乎/`。
