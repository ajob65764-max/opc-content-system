# 第一个任务提示词

第一次使用时，不建议直接做一个特别大的项目。选择一个现在就需要完成、30—90分钟内能验收的真实任务。

## 版本一：通用任务启动

复制下面的内容，把方括号替换成自己的信息：

```text
我想完成：[具体任务]。

发布平台：[小红书/抖音/公众号/视频号/知乎]
目标读者：[这份内容写给谁]
任务目的：[希望读者看完理解什么、相信什么或完成什么]
现有素材：[粘贴经历、原稿、观点、数据或参考材料]
成功标准：[什么样的结果算完成]
必须保留：[不能删改的信息]
明确不要：[不想出现的表达、结构或做法]

请先读取根目录中的 AGENTS.md，再读取 About_me 文件夹和对应平台模板。

开始正式创作前：
1. 先用一句话确认你对任务的理解；
2. 如果关键信息不足，最多向我提出5个问题；
3. 信息足够后，直接完成可发布的正式成品；
4. 按照系统规则保存到对应的 Outputs 文件夹；
5. 完成后告诉我文件位置，并检查是否有值得沉淀的模板结构。
```

## 版本二：已有原稿，改成指定平台内容

```text
请把下面的原始内容改成一份可以直接发布的[平台]内容。

目标读者：[目标读者]
核心目的：[传播/获客/建立信任/记录经验/引导领取资料]
必须保留：[真实经历、数据、观点或原话]
明确不要：[AI味、夸张收益、强营销或其他要求]

原始内容：
[粘贴原稿]

请先读取 AGENTS.md、About_me 和对应平台模板。不要只做摘要，要根据平台重新组织表达。完成后保存到对应的 Outputs 文件夹。
```

## 版本三：一个选题生成五个平台版本

```text
请围绕下面的核心选题，分别生成小红书、抖音、公众号、视频号和知乎版本。

核心选题：[选题]
核心观点：[最想表达的一句话]
目标人群：[写给谁]
现有素材：[经历、案例、原稿、数据]
希望读者采取的动作：[关注/评论/收藏/领取资料/进一步咨询/无转化要求]
必须保留：[不能改动的信息]
明确不要：[不想出现的内容]

请读取 AGENTS.md、About_me 和 Templates/一稿多平台改写模板.md。
五个平台不是简单缩写，要分别适配平台的内容结构和表达方式，并保存到各自的 Outputs 文件夹。
```

## 第一次任务示例

```text
我想写一篇小红书图文，主题是“普通上班族为什么需要搭建自己的AI工作系统”。

目标读者：白天上班、晚上做副业，但每次使用AI都要重新交代背景的人。
任务目的：让读者理解文件夹系统比收藏更多AI工具更重要。
现有素材：我每天晚上只有1—2小时做副业，经常把时间花在重复解释背景、复制内容和改格式上。
成功标准：读者看完能理解这套系统解决什么问题，并愿意收藏照着搭建。
必须保留：“人负责判断，AI负责生产，文件夹负责沉淀。”
明确不要：夸张收益、创业暴富、空泛鸡汤、强行销售。

请先读取根目录 AGENTS.md、About_me 和小红书模板。信息足够后，输出标题、封面文案、正文、话题和配图建议，并保存到 Outputs/小红书。
```
