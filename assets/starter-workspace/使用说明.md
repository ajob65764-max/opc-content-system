# 《3天搭建最小 OPC 工作系统》使用说明

这是一套面向内容创作者、上班族副业实践者和一人公司的轻量工作系统。

它不会替你决定方向，也不会承诺自动完成所有工作。它解决的是三个更具体的问题：

1. 不再每次向 AI 重新解释自己是谁；
2. 每一次内容生产都能留下可见交付物；
3. 验证过的结构能够继续复用，而不是用完就丢。

## 一、系统结构

```text
3天搭建最小OPC工作系统/
├── AGENTS.md
├── 使用说明.md
├── 第一个任务提示词.md
├── 3天执行清单.md
├── About_me/
├── Outputs/
└── Templates/
```

三个文件夹分别承担不同作用：

- `About_me/`：保存长期背景、写作偏好和业务方向；
- `Outputs/`：保存小红书、抖音、公众号、视频号、知乎的正式交付物；
- `Templates/`：保存已经整理好的平台模板和一稿多平台流程。

根目录中的 `AGENTS.md` 是系统运行规则。使用 Codex 打开整个项目文件夹后，它会按照这份文件中的要求执行任务。

## 二、第一次使用

### 第1步：解压并打开整个文件夹

不要只打开其中一个 Markdown 文件。需要把“3天搭建最小OPC工作系统”整个文件夹作为项目交给 Codex。

### 第2步：填写3个核心文件

按顺序填写：

1. `About_me/about-me.md`
2. `About_me/anti-ai-writing-style.md`
3. `About_me/my-company.md`

文件里已经准备了填写位置和示例。删除示例，替换成自己的真实情况即可。暂时无法确定的内容可以写“待补充”，不要为了填满而编造。

### 第3步：运行第一个任务

打开 `第一个任务提示词.md`，复制其中的提示词，把方括号里的内容换成自己的任务。

建议第一次只做一个真实的小任务，例如：

- 写一篇小红书图文；
- 把一段经历改成抖音口播；
- 整理一篇公众号文章；
- 把一个观点改成视频号脚本；
- 回答一个知乎问题。

### 第4步：检查保存结果

任务完成后，确认文件是否保存到了 `Outputs/` 中对应的平台目录。

例如：

```text
Outputs/小红书/2026-07-09-小红书-我的第一个OPC任务-v1.md
```

## 三、日常任务怎么发起

推荐使用这个结构：

```text
我想完成：[具体任务]
发布平台：[小红书/抖音/公众号/视频号/知乎]
目标读者：[写给谁]
使用素材：[经历、数据、原稿或参考内容]
成功标准：[什么结果算完成]
必须保留：[不能删改的信息]
明确不要：[不想出现的表达或做法]

请先按照 AGENTS.md 读取 About_me 和对应平台模板。
如果关键信息不足，先向我提问；信息足够后直接完成，并保存到对应的 Outputs 文件夹。
```

## 四、内容生产流程

```text
提供任务
→ Codex读取长期背景
→ 读取对应平台模板
→ 澄清缺失信息
→ 生成正式成品
→ 保存到Outputs
→ 检查是否值得沉淀为模板
```

不要把每次任务都做成复杂工作流。先让“背景能读取、结果能保存、结构能复用”稳定跑起来。

## 五、五个平台怎么选择

| 平台 | 更适合的内容 | 主要交付物 |
| --- | --- | --- |
| 小红书 | 经验、教程、清单、避坑、过程记录 | 标题、封面文案、正文、话题、配图建议 |
| 抖音 | 强痛点、快速判断、故事、可视化方法 | 钩子、口播、字幕、画面、结尾引导 |
| 公众号 | 完整复盘、深度教程、长期观点 | 标题、摘要、长文、文末承接 |
| 视频号 | 真人表达、专业信任、关系建立 | 口播、字幕、拍摄建议、自然引导 |
| 知乎 | 问题解答、经验论证、观点分析 | 回答标题、结论、论证、案例、总结 |

## 六、一稿多平台怎么用

当一个选题已经有完整素材时，读取 `Templates/一稿多平台改写模板.md`。

重点不是把同一篇文章缩短五次，而是保留同一个核心观点，再根据不同平台重组内容。五个平台的版本应分别保存到对应目录。

## 七、如何让系统越来越适合自己

每次出现以下情况时，更新系统：

- Codex反复写出你不喜欢的表达：补充到 `anti-ai-writing-style.md`；
- 业务方向、阶段目标或平台重点发生变化：更新 `my-company.md`；
- 某种内容结构连续两次有效：整理进对应的 `Templates/`；
- 某个模板已经不再使用：标记“停用”，不要直接删除历史版本。

## 八、常见问题

### 1. 为什么 Codex 没有使用背景文件？

确认打开的是整个项目根目录，并且 `AGENTS.md` 位于根目录。新任务开始时可明确补一句：“请先读取根目录 AGENTS.md，并按其中规则执行。”

### 2. 每次都要填写任务卡吗？

不需要。简单任务可以直接说。任务越重要、素材越多，任务卡越能减少返工。

### 3. Outputs 会不会越来越乱？

按照“日期-平台-选题-版本”的方式命名。定期把未完成稿标记为草稿，把发布稿标记为最终版。

### 4. Templates 是越多越好吗？

不是。只有真实使用过、验证有效、以后还会重复的结构，才值得保存成模板。

## 九、使用边界

- 不把账号密码、身份证件、客户隐私和未脱敏商业机密写进背景文件；
- 涉及医疗、法律、金融、政策和平台规则的时效信息，要重新核实；
- AI 可以协助生产和整理，但最终发布、事实判断和商业取舍仍由使用者负责。
