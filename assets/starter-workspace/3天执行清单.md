# 3天搭建最小 OPC 工作系统｜执行清单

目标不是三天做出复杂自动化，而是完成一个可以持续使用的最小版本。

## 第1天：让 Codex 知道你是谁

预计用时：30—60分钟。

### 必做任务

- [ ] 解压整套文件，并用 Codex 打开项目根目录；
- [ ] 阅读 `使用说明.md`；
- [ ] 填写 `About_me/about-me.md`；
- [ ] 填写 `About_me/anti-ai-writing-style.md`；
- [ ] 填写 `About_me/my-company.md`；
- [ ] 删除三个文件中的示范内容，保留自己的真实信息；
- [ ] 检查是否写入隐私、密码或不适合长期保存的信息。

### 第1天完成标准

你能够用三句话说明：

1. 我是谁；
2. 我正在做什么；
3. 我希望 AI 如何与我协作。

不要追求一次填完整。先写当前最确定的内容，后续边用边补。

## 第2天：完成第一个真实交付物

预计用时：45—90分钟。

### 必做任务

- [ ] 从五个平台中选择一个当前最重要的平台；
- [ ] 找到一个本周真实需要发布的选题；
- [ ] 打开 `第一个任务提示词.md`；
- [ ] 填写任务、读者、目的、素材和成功标准；
- [ ] 让 Codex 读取对应平台模板；
- [ ] 完成一份可以直接使用的内容；
- [ ] 将文件保存到对应的 `Outputs/` 文件夹；
- [ ] 根据发布前检查清单修改一次。

### 第2天完成标准

`Outputs/` 中至少出现一个真实成品，而不是只有计划、建议或提示词。

建议第一次任务尽量小：一篇图文、一条短视频脚本、一篇回答，或者一篇已经有素材的文章。

## 第3天：完成一次复用和沉淀

预计用时：60—120分钟。

### 必做任务

- [ ] 选择第2天已经完成的内容；
- [ ] 使用 `Templates/一稿多平台改写模板.md`；
- [ ] 至少改写成另外两个平台版本；
- [ ] 分别保存到对应的 `Outputs/` 文件夹；
- [ ] 比较不同平台的标题、开头、结构和结尾；
- [ ] 找出本次最好用的一个内容结构；
- [ ] 判断它是否值得加入 `Templates/`；
- [ ] 把新发现的写作禁区补充到 `anti-ai-writing-style.md`。

### 第3天完成标准

你已经跑通下面这条最小闭环：

```text
长期背景
→ 一个真实任务
→ 一份正式输出
→ 多平台复用
→ 模板或规则沉淀
```

## 三天之后怎么继续

每周只做三件事：

1. 至少完成一个正式输出；
2. 至少复用一次已有内容或模板；
3. 至少修正一条长期协作规则。

当这套轻量流程稳定运行后，再考虑自动发布、数据抓取、多智能体、定时任务等更复杂的自动化。
