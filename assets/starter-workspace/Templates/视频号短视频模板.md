# 视频号短视频模板

适用于真人口播、经验分享、行业判断、客户问题解答、案例复盘和私域信任内容。

视频号可以与抖音使用同一个选题，但不建议直接复制脚本。这里更重视真实表达、可信度和关系建立。

## 一、开始前需要的信息

```text
选题：
目标人群：
说话人的身份：
视频时长：
核心观点：
真实经历或案例：
希望建立的信任：
希望观众采取的动作：
必须保留：
明确不要：
```

## 二、推荐结构

```text
点明目标人群和问题
→ 给出清楚判断
→ 用经历、案例或方法解释
→ 说明适用边界
→ 自然结束或给出下一步
```

### 开头

开头要直接，但不必刻意制造冲突。可以使用：

- “如果你白天上班、晚上做副业，这个问题大概率遇到过。”
- “我最近没有继续找新的AI工具，反而先整理了自己的文件夹。”
- “很多人用AI效率还是不高，问题可能不在模型。”

### 中段

- 像向一个具体的人解释，不像发表演讲；
- 保留真实语气和停顿；
- 一个视频集中讲一个主要判断；
- 用具体例子建立可信度。

### 结尾

优先给观众一个能立即执行的小动作。需要承接关注或资料时，先说明继续关注能获得什么。

## 三、输出格式

```text
视频标题：
封面文案：
目标时长：

完整口播：
[正文]

重点字幕：
1.
2.
3.

拍摄建议：
[景别、动作、辅助画面]

发布文案：
[简短说明]
```

## 四、可直接复制的提示词

```text
请根据下面的信息，制作一条适合视频号发布的真人口播脚本。

选题：[填写]
目标人群：[填写]
说话人身份：[填写]
视频时长：[填写]
核心观点：[填写]
真实经历或案例：[填写]
希望建立的信任：[填写]
希望观众采取的动作：[填写]
必须保留：[填写]
明确不要：[填写]

请先读取 About_me 和本模板。输出视频标题、封面文案、完整口播、重点字幕、拍摄建议和发布文案。

表达要像真人面对镜头说话。不要直接套用抖音式强冲突，不要空泛说教，不要强迫互动。完成后保存到 Outputs/视频号。
```

## 五、简短示例

> 如果你白天上班、晚上做副业，真正稀缺的不是AI工具，而是那一两个小时。
>
> 我以前每次打开AI，都要重新解释自己的背景。后来我建了三个文件夹：一个放我是谁，一个放做出来的东西，一个放以后还能再用的模板。
>
> 它没有替我做决定，但确实减少了很多重复工作。你也可以先从这三个文件夹开始，不用一上来搭复杂自动化。

## 六、发布前检查

- [ ] 说话人的身份和目标人群清楚；
- [ ] 口播读起来像真人表达；
- [ ] 一个视频只讲一个主要判断；
- [ ] 有经历、例子或具体方法支撑；
- [ ] 没有强行制造焦虑；
- [ ] 结尾自然并且能够执行；
- [ ] 已保存到 `Outputs/视频号/`。
