# 小红书图文模板

适用于经验分享、教程步骤、避坑清单、工具实测、项目复盘和资料型内容。

## 一、开始前需要的信息

```text
选题：
目标读者：
读者当前处境：
核心观点：
真实经历或素材：
希望读者采取的动作：
必须保留：
明确不要：
```

缺少真实素材时先提问，不编造使用经历、收入、案例和效果。

## 二、标准输出结构

### 1. 标题备选

输出5个不同角度的标题，优先控制在移动端容易完整显示的长度。

可选角度：

- 痛点判断型；
- 真实经历型；
- 结果展示型；
- 方法清单型；
- 反常识观点型。

标题要具体，不使用“震惊”“所有人都应该”“错过后悔”等过度表达。

### 2. 封面文案

- 主标题：一句话讲清内容价值；
- 副标题：补充对象、场景或结果；
- 封面文字尽量少，不把正文全部堆到封面。

### 3. 正文

推荐结构：

```text
真实场景或痛点
→ 我为什么开始解决
→ 核心判断
→ 3—5个具体步骤或发现
→ 适用边界
→ 自然行动建议
```

正文要求：

- 开头尽快进入具体问题；
- 一段不宜过长；
- 多写动作、判断标准和真实变化；
- 需要列步骤时，每一步都要能执行；
- 结尾不重复全文，不强行升华。

### 4. 发布辅助

- 3—8个相关话题关键词；
- 首图和内页配图建议；
- 如需评论引导，给出自然、低门槛的问法。

## 三、可直接复制的提示词

```text
请根据下面的信息，制作一篇可以直接发布的小红书图文。

选题：[填写]
目标读者：[填写]
读者当前处境：[填写]
核心观点：[填写]
真实素材：[填写]
希望读者采取的动作：[填写]
必须保留：[填写]
明确不要：[填写]

请先读取 About_me 和本模板。输出：
1. 5个不同角度的标题；
2. 3组封面主副标题；
3. 完整正文；
4. 话题关键词；
5. 配图建议；
6. 发布前检查结果。

不要编造经历，不要写夸张收益，不要使用空泛鸡汤。完成后保存到 Outputs/小红书。
```

## 四、简短示例

选题：普通上班族为什么需要自己的AI工作系统。

封面主标题：

> 我用3个文件夹，结束了每次从零用AI

开头示例：

> 我晚上真正能拿来做副业的时间，通常只有一两个小时。以前每次打开AI，前20分钟都在重新解释：我是谁、想做什么、讨厌什么风格。后来我没有继续找新工具，而是先建了3个文件夹。

## 五、发布前检查

- [ ] 标题和正文表达的是同一件事；
- [ ] 前两段已经说明目标人群和问题；
- [ ] 有真实信息，不是只有正确的观点；
- [ ] 步骤能够执行；
- [ ] 没有编造结果和夸张承诺；
- [ ] 结尾引导与正文自然衔接；
- [ ] 已保存到 `Outputs/小红书/`。
