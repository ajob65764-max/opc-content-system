# 抖音短视频模板

适用于30—90秒口播、知识讲解、同城内容、经验分享、案例拆解和问题诊断。

## 一、开始前需要的信息

```text
选题：
目标人群：
视频时长：
核心问题：
最重要的结论：
真实素材或案例：
拍摄条件：真人口播/画外音/图文混剪
希望观众采取的动作：
必须保留：
明确不要：
```

## 二、标准脚本结构

### 1. 前3秒

开头只承担一个任务：让目标观众确认“这条内容和我有关”。

常用方式：

- 直接点人群：“每天只有一两个小时做副业的人……”
- 直接点问题：“你每次用AI，是不是都要重新解释一遍自己？”
- 直接给判断：“普通人先别急着搭复杂AI工作流。”
- 直接展示变化：“我只建了3个文件夹，后面的内容终于不再散了。”

不要先自我介绍，不要用“今天给大家分享”，不要铺垫十几秒才进入主题。

### 2. 中段内容

推荐结构：

```text
问题场景
→ 为什么会这样
→ 解决方法或判断标准
→ 一个具体例子
```

口播短句化。一句话太长时拆开。每8—15秒安排一次信息变化、字幕重点或画面变化。

### 3. 结尾

结尾可以选择：

- 给出下一步动作；
- 回答开头的问题；
- 提出低门槛问题；
- 自然承接资料、关注或咨询。

不要无条件使用“评论区打666”“赶紧点赞收藏”。引导必须与内容相关。

## 三、输出格式

| 时间 | 口播 | 屏幕字幕 | 画面/动作 |
| --- | --- | --- | --- |
| 0—3秒 | [钩子] | [核心大字] | [镜头动作] |
| 3—15秒 | [问题] | [关键词] | [辅助画面] |
| 15—45秒 | [方法] | [步骤字幕] | [演示/切镜] |
| 45—60秒 | [结尾] | [行动提示] | [收尾镜头] |

```text
请根据下面的信息，制作一条可以直接拍摄的抖音短视频脚本。

选题：[填写]
目标人群：[填写]
视频时长：[填写]
核心问题：[填写]
核心结论：[填写]
真实素材：[填写]
拍摄条件：[填写]
希望观众采取的动作：[填写]
必须保留：[填写]
明确不要：[填写]

请先读取 About_me 和本模板。输出：
1. 5个前3秒钩子；
2. 1版完整口播稿；
3. 按时间拆分的口播、屏幕字幕和画面建议；
4. 视频标题和发布文案；
5. 发布前检查结果。

口播必须自然、短句化，不编造案例，不使用无法兑现的承诺。完成后保存到 Outputs/抖音。
```

## 五、简短示例

```text
你每次用AI，是不是都要重新解释一遍自己？

我以前也是。
晚上只有一两个小时做副业，前20分钟都在交代背景、改格式。

后来我只建了3个文件夹。
一个放我的长期背景，一个放正式输出，一个放验证过的模板。

AI没有突然变聪明。
但它终于不再每次从零开始。
```

## 六、发布前检查

- [ ] 前3秒已经点出人群、问题或结果；
- [ ] 没有无效自我介绍；
- [ ] 口播读出来顺畅；
- [ ] 屏幕字幕不是整段复制口播；
- [ ] 画面建议符合实际拍摄条件；
- [ ] 没有虚假焦虑和绝对承诺；
- [ ] 已保存到 `Outputs/抖音/`。
