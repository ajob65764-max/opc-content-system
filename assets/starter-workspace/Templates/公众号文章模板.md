# 公众号文章模板

适用于深度教程、项目复盘、经验总结、观点文章、资料包承接和长期内容沉淀。

## 一、开始前需要的信息

```text
文章主题：
目标读者：
读者看完要理解或完成什么：
核心观点：
个人经历或案例：
需要引用的资料：
预计篇幅：
文末是否承接资料或产品：
必须保留：
明确不要：
```

## 二、推荐文章结构

### 1. 标题

先给5—10个标题备选，覆盖：

- 具体结果；
- 真实经历；
- 明确问题；
- 方法或清单；
- 核心观点。

标题不使用与正文不符的夸张结果。

### 2. 摘要

用50—120字说明：文章写给谁、解决什么问题、读者能获得什么。

### 3. 开头

从真实场景、具体问题、一次经历或明确判断进入。开头不要大段解释行业背景。

### 4. 正文

推荐结构：

```text
问题是怎么出现的
→ 我以前怎么处理
→ 为什么原来的方式不够用
→ 我现在的核心判断
→ 具体做法
→ 实际使用场景
→ 适用边界和下一步
```

每个小标题应该推进内容，而不是换句话重复标题。

### 5. 文末承接

如果需要承接资料或产品，应说明：

- 它适合谁；
- 具体包含什么；
- 能帮助完成什么；
- 如何获取。

不制造虚假稀缺，不承诺无法保证的结果。

## 三、可直接复制的提示词

```text
请根据下面的信息，制作一篇可以直接发布的公众号文章。

主题：[填写]
目标读者：[填写]
阅读目标：[填写]
核心观点：[填写]
个人经历或案例：[填写]
参考资料：[填写]
预计篇幅：[填写]
文末承接：[填写]
必须保留：[填写]
明确不要：[填写]

请先读取 About_me 和本模板。开始前检查素材是否足够，不足时先提问。

输出：
1. 8个标题备选；
2. 文章摘要；
3. 完整正文；
4. 文末承接；
5. 封面文案和配图建议；
6. 发布前检查结果。

保留真实经历和判断过程，不编造案例，不空泛升华。完成后保存到 Outputs/公众号。
```

## 四、简短示例结构

```text
标题：我用3个文件夹，搭了一个最小一人公司工作系统

开头：
我现在最需要的不是多一个AI聊天窗口，而是一套能长期沉淀背景、输出和模板的工作系统。

第一部分：为什么每次用AI都像从零开始
第二部分：Codex为什么适合放进项目文件夹
第三部分：3个文件夹分别解决什么问题
第四部分：真正改变输出质量的3个文件
第五部分：一个任务应该怎么跑
结尾：先跑通最小闭环，再谈复杂自动化
```

## 五、发布前检查

- [ ] 标题没有夸大正文结果；
- [ ] 开头不是空泛背景介绍；
- [ ] 每个小标题都在推进内容；
- [ ] 个人经历和事实没有被擅自改写；
- [ ] 方法有执行步骤和适用边界；
- [ ] 文末承接与正文自然相关；
- [ ] 已保存到 `Outputs/公众号/`。
